# End-to-End UX-аудит и перепроектирование критических сценариев

## 1) Цель и ожидаемый бизнес-эффект

Сфокусироваться на 2–3 ключевых сценариях (например: первый запуск, заявка/покупка, повторное действие) и:
- сократить путь до целевого действия;
- убрать UX-тупики и избыточные решения «в лоб»;
- снизить когнитивную нагрузку и количество ошибок.

**Целевой результат (90 дней):**
- рост конверсии ключевого сценария;
- снижение drop-off на критических шагах;
- сокращение времени выполнения задачи.

---

## 2) Объем работ (что делаем)

### A. CJM (Customer Journey Map)

Собираем CJM отдельно для:
- новых пользователей;
- возвращающихся пользователей;
- пользователей с незавершенным целевым действием.

Для каждого этапа фиксируем:
- контекст и мотивацию;
- задачи пользователя;
- эмоции/уровень уверенности;
- барьеры и триггеры ухода;
- точки влияния интерфейса и контента.

**Выход:** карта pain points + список гипотез по улучшению каждого этапа.

### B. Анализ воронок

Строим воронки по критическим сценариям:
1. Вход в сценарий;
2. Ключевые шаги;
3. Успешное завершение.

Для каждого шага анализируем:
- drop-off rate;
- time-to-step;
- возвраты назад/циклы;
- ошибки форм/валидаций;
- device/source/segment split.

**Выход:** приоритизированный список шагов с максимальной потерей конверсии.

### C. Интервью (качественные исследования)

Проводим 12–18 интервью:
- 40% новые пользователи;
- 40% пользователи с частичным успехом;
- 20% пользователи, не завершившие сценарий.

Фокус интервью:
- почему пользователь начинает сценарий;
- где возникает неопределенность;
- какие термины/элементы непонятны;
- что заставляет откладывать или уходить.

**Выход:** карта ментальных моделей и «языка пользователя» для переработки UX-копирайта.

### D. Usability-тесты (модерируемые + немодерируемые)

- 2 раунда по 8–10 респондентов;
- тестируем текущий и прототипируемый варианты;
- сценарии: onboarding, ключевая форма, CTA, навигация.

Метрики:
- task success rate;
- time on task;
- error rate;
- misclicks;
- субъективная сложность (SEQ).

**Выход:** подтвержденные UX-проблемы с приоритетом Severity × Frequency.

---

## 3) Метод перепроектирования критических сценариев

1. **As-is blueprint** (текущий поток, шаги, данные, состояния, ошибки).
2. **Удаление/слияние шагов** (каждый шаг проходит фильтр «создает ли ценность?»).
3. **Прогрессивное раскрытие** (убираем перегруз в ранних шагах).
4. **Анти-тупики**:
   - всегда есть следующий понятный action;
   - всегда есть путь назад без потери данных;
   - ошибки объясняют «что делать дальше».
5. **Снижение когнитивной нагрузки**:
   - единые паттерны CTA;
   - predictable labels;
   - chunking информации;
   - prefill и smart defaults.
6. **To-be flow + прототипы** (low-fi → hi-fi).
7. **Пилот/feature flag** для безопасного релиза.

---

## 4) KPI и целевые метрики

### Северная звезда (North Star для UX)
- Completion Rate ключевого сценария.

### Основные KPI
- Conversion Rate ключевого сценария;
- Drop-off Rate на топ-3 проблемных шагах;
- Median Time to Completion;
- Task Success Rate в usability-тестах.

### Guardrail-метрики
- рост ошибок форм;
- рост обращений в поддержку по сценарию;
- рост bounce rate на входном экране.

---

## 5) План внедрения (12 недель)

- **Недели 1–2:** аналитическая подготовка, событийная карта, baseline KPI.
- **Недели 3–4:** CJM + интервью + анализ воронок.
- **Недели 5–6:** генерация и приоритизация гипотез, быстрые прототипы.
- **Недели 7–8:** usability-тесты, доработка решений.
- **Недели 9–10:** A/B-пилоты на ключевых шагах.
- **Недели 11–12:** релиз победивших вариантов, пост-анализ.

---

## 6) Артефакты на выходе

- CJM по ключевым сегментам;
- карта воронок и top drop-off причин;
- backlog UX-гипотез (ICE/RICE);
- новая схема критических user flows;
- прототипы с обоснованием решений;
- отчет «до/после» по KPI.
