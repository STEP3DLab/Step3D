# Step3D Content Guide

## Где менять контент
- Структура страницы и тексты: `index.html`
- Данные по кейсам (карточки + детальные модалки): `assets/js/cases.js`
- Логика интерфейса (навигация, фильтры, FAQ, форма): `assets/js/main.js`
- Базовые токены и типографика: `assets/css/styles.css`
- Компоненты и состояния: `assets/css/components.css`
- Адаптив: `assets/css/responsive.css`

## Что важно перед публикацией
1. Используйте только относительные пути (GitHub Pages).
2. Проверяйте якоря в меню, фильтры кейсов, модалку кейса и форму контакта.
3. Для новых кейсов соблюдайте структуру объекта из `assets/js/cases.js`.
4. Не оставляйте служебные тексты и временные заглушки в интерфейсе.
