# Step3D Content Guide

## Где менять контент
- Структура страницы и тексты: `index.html`
- Данные по кейсам (карточки + детальные модалки): `assets/js/cases.js`
- Данные по историям (story viewer): `assets/js/stories.js`
- Логика интерфейса и runtime-валидация данных: `assets/js/main.js`
- Базовые токены и типографика: `assets/css/styles.css`
- Компоненты и состояния: `assets/css/components.css`
- Адаптив: `assets/css/responsive.css`

## Минимальная схема данных

### Кейсы (`assets/js/cases.js`)
Обязательные поля:
- `id` — уникальный строковый идентификатор (используется в `data-case-id`).
- `title` — заголовок карточки/модалки.
- `categoryLabel` — тип кейса (верхняя метка).
- `type` — массив тегов для фильтра (`scan`, `reverse`, `print`, `prototype`).
- `problem`, `solution`, `result`, `taskType`, `output` — текстовые блоки модалки.

Optional-поля:
- `timeline` — срок. Если пусто, в UI отображается `—`.
- `budget` — бюджет. Если пусто, в UI отображается `—`.

### Истории (`assets/js/stories.js`)
Формат: `window.STEP3D_STORIES = []` (массив объектов).

Обязательные поля истории:
- `id` — уникальный строковый идентификатор (совпадает с `data-story-id` в HTML).
- `title` — заголовок истории.
- `slides` — массив минимум из 1 валидного слайда.

Optional-поля истории:
- `subtitle` — подпись над заголовком (по умолчанию: `История направления`).

Обязательные поля слайда:
- `title` — заголовок слайда.
- `text` — основной текст.

Optional-поля слайда:
- `media` — объект `{ type, src }`, где `type` = `image` или `video`.
  - Если `media` неполный или с неизвестным `type`, он будет проигнорирован (без падения UI).

## Runtime-валидация (как работает)
- При загрузке данные историй валидируются в `assets/js/main.js`.
- Невалидные истории/слайды пропускаются, а в консоль пишется `console.warn` с причиной.
- UI показывает только валидные элементы; из-за неполных записей модалка не падает.
- Для optional-полей кейсов (`timeline`, `budget`) используется безопасный fallback `—`.

## Чеклист перед публикацией
1. Используйте только относительные пути (GitHub Pages).
2. Проверяйте якоря в меню, фильтры кейсов, модалки кейса/историй и форму контакта.
3. Для новых кейсов соблюдайте схему из `assets/js/cases.js`.
4. Для новых историй соблюдайте схему из `assets/js/stories.js`.
5. Не оставляйте служебные тексты и временные заглушки в интерфейсе.
