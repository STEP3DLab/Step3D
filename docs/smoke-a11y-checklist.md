# Smoke A11Y checklist

Минимальный smoke-чек перед релизом для быстрых ручных проверок доступности.

## 1) Tab order / keyboard-only

- [ ] `Tab` проходит по интерактивным элементам в логичном порядке (header → main → footer).
- [ ] Skip-link (`Перейти к контенту`) доступен с клавиатуры и переводит фокус в `#main`.
- [ ] Для фильтров кейсов (`role="tablist"`) работают `ArrowLeft` / `ArrowRight` / `Home` / `End`.
- [ ] В FAQ кнопки открываются/закрываются по `Enter` и `Space`, фокус между вопросами перемещается по `ArrowUp` / `ArrowDown`.
- [ ] При открытом мобильном меню фокус не выходит за пределы меню (focus trap), `Esc` закрывает меню.
- [ ] После открытия деталей кейса фокус переводится на заголовок детали; при закрытии/`Back` в истории возвращается к исходному триггеру.

## 2) Screen reader labels / semantics

- [ ] У всех интерактивных элементов есть доступные имена (`aria-label` или видимый текст).
- [ ] Связка вкладок кейсов полная: `role="tab"` + `aria-controls` + `role="tabpanel"` + `aria-labelledby`.
- [ ] У FAQ-кнопок есть `aria-controls`, у панелей — `aria-labelledby`.
- [ ] Основные landmark-блоки корректны (`header`, `nav`, `main`, `section`, `footer`).

## 3) ARIA state transitions

- [ ] Вкладки кейсов корректно переключают `aria-selected` и roving `tabindex` (`0` только у активной вкладки).
- [ ] FAQ синхронизирует `aria-expanded` кнопок и `hidden` / `aria-hidden` панелей.
- [ ] Кнопка мобильного меню синхронизирует `aria-expanded` в состояниях open/close.
- [ ] Динамические блоки с `aria-live` обновляются без "зависших" или дублирующихся анонсов.

## 4) Focus visibility

- [ ] Для ссылок, кнопок, полей, табов, story-кнопок и чипов виден единый `:focus-visible` стиль.
- [ ] Фокус-индикатор контрастен на светлой и тёмной теме.
